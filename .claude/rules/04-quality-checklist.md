# 质量检查清单

## 文档创建/更新检查

### Markdown 文档检查

#### 结构完整性
- [ ] 文件命名符合规范（`项目名称-YYYYMMDD.md`）
- [ ] 包含一级标题（唯一）
- [ ] 包含元信息（创建时间、负责人、状态）
- [ ] 包含最后更新时间（文档底部）
- [ ] 标题层级正确（# ## ### 不超过三级）

#### 格式规范
- [ ] 日期格式一致（`YYYY-MM-DD`）
- [ ] 任务列表格式正确（`- [ ]` 或 `- [x]`）
- [ ] 表格格式完整（有表头和分隔线）
- [ ] 代码块使用正确的语法标记
- [ ] 链接使用相对路径且有效

#### 内容质量
- [ ] 项目目标清晰明确
- [ ] 范围定义具体
- [ ] 里程碑可测量、有截止日期
- [ ] 风险和依赖明确列出
- [ ] 无拼写错误和语法错误

### JSON 注册表检查

#### 格式正确性
- [ ] 文件命名符合规范（`PROJ-NNN-简称.json`）
- [ ] JSON 格式正确（可解析）
- [ ] 无语法错误（逗号、引号、括号）

#### 必填字段
- [ ] `id` - 项目唯一标识符
- [ ] `name` - 项目名称
- [ ] `status` - 项目状态（active/completed/paused/cancelled）
- [ ] `created_date` - 创建日期（YYYY-MM-DD）
- [ ] `updated_date` - 更新日期（YYYY-MM-DD）
- [ ] `owner` - 负责人
- [ ] `documents.project` - 项目主文档路径

#### 数据有效性
- [ ] 项目 ID 唯一（不与其他项目重复）
- [ ] 日期格式正确（YYYY-MM-DD）
- [ ] 状态值有效（active/completed/paused/cancelled）
- [ ] 文档路径存在且正确
- [ ] `updated_date` ≥ `created_date`

### 一致性检查

#### 跨文档一致性
- [ ] JSON 中的项目名称与 Markdown 标题一致
- [ ] JSON 中的文档路径与实际文件路径一致
- [ ] JSON 中的状态与 Markdown 元信息一致
- [ ] JSON 中的里程碑与 Markdown 内容一致

#### 时间一致性
- [ ] `updated_date` 与最新修改时间匹配
- [ ] 里程碑日期按时间顺序排列
- [ ] 已完成里程碑的日期 ≤ 当前日期

## 项目生命周期检查

### 项目创建时
- [ ] 从标准模板创建
- [ ] 同时创建 Markdown 和 JSON
- [ ] 元信息完整填写
- [ ] Git 提交消息规范

### 项目进行中
- [ ] 定期更新进度
- [ ] 及时标记完成的里程碑
- [ ] 更新 `updated_date`
- [ ] 记录重要决策和变更

### 项目归档前
- [ ] 所有里程碑已完成或明确取消
- [ ] 创建回顾文档（推荐）
- [ ] 移动到 `archived/` 目录
- [ ] 更新注册表状态为 `completed`

## 定期维护检查

### 每周检查
- [ ] 查看即将到期的里程碑
- [ ] 检查长时间未更新的项目
- [ ] 验证活跃项目数量合理

### 每月检查
- [ ] 归档已完成项目
- [ ] 审查暂停项目是否需要取消
- [ ] 检查文档质量和一致性

### 每季度检查
- [ ] 清理冗余文档
- [ ] 优化模板和规范
- [ ] 总结经验教训

## AI 助手自检清单

### 创建文档后
- [ ] 验证文件命名正确
- [ ] 验证 JSON 可解析
- [ ] 验证所有必填字段已填写
- [ ] 验证路径存在且一致

### 更新文档后
- [ ] 同时更新 Markdown 和 JSON
- [ ] 更新 `updated_date`
- [ ] 验证数据一致性
- [ ] 提示用户下一步行动

### 回答问题前
- [ ] 优先读取项目注册表
- [ ] 验证信息来源可靠
- [ ] 提供具体引用路径

---

**使用建议**：
- 创建/更新文档时，逐项核对清单
- AI 助手应自动执行检查
- 人工审查时使用清单避免遗漏
