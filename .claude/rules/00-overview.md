# 项目规则概述

## 规则文件组织

`.claude/rules/` 目录包含模块化的项目规则，按主题分类：

- `00-overview.md` - 本文件，规则体系概述
- `01-project-context.md` - 项目背景与核心理念
- `02-documentation-standards.md` - 文档规范与格式要求
- `03-workflow-guidelines.md` - 工作流程与操作规范
- `04-quality-checklist.md` - 质量检查清单
- `05-team-conventions.md` - 团队协作约定

## 规则优先级

1. **项目级规则** (`.claude/rules/*.md`) - 最高优先级
2. **根目录规则** (`AGENTS.md`) - 次优先级
3. **默认 AI 行为** - 基础优先级

## 使用指南

### 对 AI 助手
- 优先读取 `.claude/rules/` 中的规则
- 遵循规则中的约定和标准
- 规则冲突时，优先级高的规则覆盖低优先级规则

### 对团队成员
- 所有规则通过 Git 版本控制
- 修改规则需提交 PR 并说明理由
- 新成员应首先阅读本目录下所有规则文件

## 规则更新流程

1. 识别需求或问题
2. 提出规则修改建议
3. 团队讨论达成共识
4. 更新相应规则文件
5. 提交 Git 并通知团队
