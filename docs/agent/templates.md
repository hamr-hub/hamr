# 文档模板说明

## 可用模板列表

### 1. 项目文档模板
**文件**: `templates/project-template.md`

用于创建新项目的完整文档。

**使用场景**:
- 启动新项目
- 制定项目计划

**包含章节**:
- 项目概述（目标、背景、价值）
- 项目范围（包含/不包含）
- 关键里程碑
- 团队成员与职责
- 风险与依赖
- 资源需求

---

### 2. 迭代计划模板
**文件**: `templates/sprint-template.md`

用于敏捷开发的迭代计划。

**使用场景**:
- 开始新的迭代/冲刺
- 制定双周或单周开发计划

**包含章节**:
- 迭代目标
- 任务列表（含优先级、负责人、状态）
- 风险与阻碍
- 每日进度跟踪

---

### 3. 产品路线图模板
**文件**: `templates/roadmap-template.md`

用于规划产品方向和长期目标。

**使用场景**:
- 制定季度/年度规划
- 向团队或管理层展示产品方向

**包含章节**:
- 愿景与目标
- 时间线（季度划分）
- 功能优先级
- 依赖与前置条件

---

### 4. 回顾会议模板
**文件**: `templates/retrospective-template.md`

用于迭代或项目结束后的复盘。

**使用场景**:
- 迭代结束回顾
- 项目完结复盘
- 团队经验总结

**包含章节**:
- 做得好的方面
- 需要改进的地方
- 行动计划
- 数据指标（可选）

---

### 5. 技术提案模板
**文件**: `templates/proposal-template.md`

用于技术方案选型或架构设计文档。

**使用场景**:
- 技术选型决策
- 架构设计讨论
- 需要评审的技术方案

**包含章节**:
- 问题描述
- 备选方案对比
- 推荐方案详细说明
- 实施计划
- 风险评估

---

### 6. 周报/月报模板
**文件**: `templates/report-template.md`

用于定期进度汇报。

**使用场景**:
- 项目周报
- 团队月报
- 管理层汇报

**包含章节**:
- 本周/月完成工作
- 下周/月计划
- 风险与问题
- 关键指标

---

## 使用方法

### 创建新文档
```bash
# 从模板创建项目文档
cp templates/project-template.md projects/active/新项目-20260304.md

# 从模板创建迭代计划
cp templates/sprint-template.md execution/sprints/sprint-01-20260304.md

# 从模板创建路线图
cp templates/roadmap-template.md planning/roadmaps/产品A-roadmap-2026Q1.md
```

### 自定义模板
如果标准模板不满足需求，可以：
1. 复制现有模板
2. 根据项目特点修改章节
3. 保存为新模板供后续使用

### 模板维护
- 从实际使用中总结经验，定期优化模板
- 团队共识后更新模板
- 保留模板变更历史（通过 Git）

## 模板最佳实践

### 填写指南
- **必填项**：用 `**` 标记
- **选填项**：用 `(可选)` 标注
- **示例内容**：用 `<!-- 示例：... -->` 提示

### 元信息
每个模板顶部应包含：
```markdown
---
模板名称: XXX
适用场景: XXX
最后更新: YYYY-MM-DD
---
```

### 灵活使用
- 模板是指导而非强制
- 根据实际情况调整章节
- 小项目可简化内容
