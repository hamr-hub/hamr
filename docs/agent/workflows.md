## 工作流 1: 创建新项目

### 触发条件
用户说："帮我创建一个新项目"，或明确提出项目需求

### 操作步骤

**步骤 1: 收集信息**
询问用户以下信息：
- 项目名称（必填）
- 项目目标（必填）
- 负责人（选填）
- 预计完成时间（选填）
- 优先级（选填，默认：中）

**步骤 2: 分配项目 ID**
```
查看现有项目注册表，获取最大 ID 号
新项目 ID = 最大 ID + 1
格式：PROJ-NNN（三位数字，不足补零）
```

**步骤 3: 创建项目主文档**
```bash
# 从模板创建
cp templates/project-template.md projects/active/项目名称-YYYYMMDD.md

# 填写以下信息
- 项目名称
- 创建时间（当天日期）
- 项目负责人
- 当前状态（🟢 进行中）
- 优先级
- 项目目标
```

**步骤 4: 创建项目注册表**
```bash
# 从模板创建
cp projects/registry/PROJ-000-template.json projects/registry/PROJ-NNN-简称.json

# 填写以下字段
{
  "id": "PROJ-NNN",
  "name": "项目全称",
  "status": "active",
  "priority": "high/medium/low",
  "created_date": "YYYY-MM-DD",
  "updated_date": "YYYY-MM-DD",
  "owner": "负责人",
  "tags": [...],
  "documents": {
    "project": "projects/active/项目名称-YYYYMMDD.md",
    ...
  },
  "milestones": [...],
  "description": "项目简介"
}
```

**步骤 5: 询问是否创建关联文档**
- 是否需要产品路线图？
- 是否需要技术提案？
- 是否需要评审文档？

**步骤 6: 确认并总结**
向用户展示：
- 已创建的文档路径
- 项目 ID
- 下一步建议（填写项目目标、里程碑等）

### 示例对话

```
用户：帮我创建一个用户认证系统的项目

AI：好的，我来帮你创建用户认证系统项目。请提供以下信息：

1. 项目目标是什么？（例如：实现安全的用户登录注册功能）
2. 项目负责人是谁？
3. 预计什么时候完成？
4. 优先级如何？（高/中/低）

用户：目标是实现 OAuth2.0 登录，我负责，6月底完成，高优先级

AI：收到！我已创建以下文档：

📄 项目文档：projects/active/用户认证系统-20260304.md
📋 注册表：projects/registry/PROJ-002-用户认证.json

项目 ID：PROJ-002
状态：进行中
优先级：高

是否需要我同时创建：
- 产品路线图（规划实施阶段）
- 技术提案（OAuth2.0 方案设计）

下一步建议：
1. 补充项目范围和关键里程碑
2. 确定团队成员和职责
3. 评估技术风险
```

## 工作流 2: 查找项目信息

### 触发条件
用户问："XXX 项目怎么样了？"、"有哪些项目？"、"XX 项目的文档在哪？"

### 操作步骤

**步骤 1: 确定查询范围**
- 查询单个项目：继续步骤 2
- 查询所有项目：跳到步骤 4
- 查询特定状态项目：跳到步骤 5

**步骤 2: 搜索项目（单个）**
```bash
# 方式 1: 通过项目名称搜索
find projects/registry -name "*关键词*.json"

# 方式 2: 通过内容搜索
grep -l "项目名称" projects/registry/*.json

# 方式 3: 直接读取（如果知道 ID）
read projects/registry/PROJ-NNN-简称.json
```

**步骤 3: 提取关键信息**
从 JSON 注册表中读取：
- 项目名称、状态、优先级
- 负责人、创建/更新时间
- 里程碑及其状态
- 文档路径

**步骤 4: 列出所有项目（如果用户要求）**
```bash
# 读取所有 JSON 文件
find projects/registry -name "PROJ-*.json" -not -name "*template*"

# 提取摘要信息
jq '{id, name, status, priority, owner}' projects/registry/PROJ-*.json
```

**步骤 5: 按状态筛选（如果用户指定）**
```bash
# 活跃项目
jq 'select(.status == "active")' projects/registry/*.json

# 已完成项目
jq 'select(.status == "completed")' projects/registry/*.json

# 高优先级项目
jq 'select(.priority == "high")' projects/registry/*.json
```

**步骤 6: 总结回复**
以清晰的格式展示：
- 项目基本信息
- 当前状态和进度
- 关键里程碑
- 如需详细信息，提供文档路径

### 示例对话

```
用户：域名规划项目进展怎么样？

AI：正在查询域名规划项目...

📊 项目信息
- 项目ID：PROJ-001
- 名称：域名规划与基础设施建设
- 状态：🟢 进行中
- 优先级：高
- 负责人：待定
- 最后更新：2026-03-04

🎯 里程碑进度
✅ 域名规划方案完成（2026-03-04）
🟢 Phase 1 子域名注册（2026-03-15，进行中）
⏳ 官网上线（2026-04-15，待开始）
⏳ 文档站上线（2026-04-15，待开始）
⏳ 核心应用上线（2026-05-30，待开始）
⏳ Phase 1 完成（2026-06-30，待开始）

📄 相关文档
- 项目文档：projects/active/域名规划与基础设施-20260304.md
- 技术提案：planning/proposals/域名分配详细清单-20260304.md
- 产品路线图：planning/roadmaps/域名基础设施-roadmap-2026Q2.md

是否需要查看详细内容？
```

## 工作流 3: 更新项目进度

### 触发条件
用户说："XX 里程碑完成了"、"更新项目进度"、"标记任务完成"

### 操作步骤

**步骤 1: 定位项目**
使用工作流 2 找到对应项目的注册表和主文档

**步骤 2: 读取当前状态**
```bash
# 读取项目文档
read projects/active/项目名称-YYYYMMDD.md

# 读取注册表
read projects/registry/PROJ-NNN-简称.json
```

**步骤 3: 确认更新内容**
询问用户：
- 完成了哪个里程碑/任务？
- 当前遇到什么问题？
- 下一步计划做什么？

**步骤 4: 更新项目文档**
在 Markdown 文档中更新：
```markdown
## 📈 进度跟踪

### 当前进度
- 整体进度：30% → 50%
- 最后更新：2026-03-04 → 2026-03-10

### 本周进展
- 完成：Phase 1 子域名注册
- 进行中：官网开发
- 遇到的问题：无

### 下周计划
- 完成官网首页设计
- 开始文档站开发
```

**步骤 5: 更新注册表**
修改 JSON 文件：
```json
{
  "updated_date": "2026-03-10",
  "milestones": [
    {
      "name": "Phase 1 子域名注册",
      "date": "2026-03-15",
      "status": "completed"  // pending → completed
    }
  ]
}
```

**步骤 6: 检查一致性**
确保：
- Markdown 和 JSON 中的状态一致
- `updated_date` 已更新
- 进度百分比合理

**步骤 7: 确认并建议**
向用户：
- 确认更新成功
- 显示下一个待完成的里程碑
- 提醒即将到期的任务

### 示例对话

```
用户：域名注册完成了

AI：好的，我来更新域名规划项目的进度。

✅ 已完成里程碑：Phase 1 子域名注册（计划：2026-03-15，实际：2026-03-10）

我已更新：
- 项目文档：标记里程碑为已完成
- 注册表：更新状态和日期
- 整体进度：15% → 30%

🎯 下一个里程碑
⏳ 官网上线（2026-04-15，还剩 36 天）

建议：
1. 开始官网设计和开发
2. 并行推进文档站建设
3. 准备服务器环境

有什么问题或阻碍吗？
```

## 工作流 4: 项目归档

### 触发条件
用户说："XX 项目完成了"、"归档 XX 项目"、项目状态变更为已完成/已取消

### 操作步骤

**步骤 1: 确认归档原因**
询问用户：
- 项目是完成了还是取消了？
- 是否需要创建回顾文档？

**步骤 2: 移动项目文档**
```bash
# 添加状态后缀
mv projects/active/项目名称-20260304.md \
   projects/archived/项目名称-20260304-completed.md

# 或者取消状态
mv projects/active/项目名称-20260304.md \
   projects/archived/项目名称-20260304-cancelled.md
```

**步骤 3: 更新注册表**
```json
{
  "status": "completed",  // active → completed
  "updated_date": "YYYY-MM-DD",
  "documents": {
    "project": "projects/archived/项目名称-20260304-completed.md",
    ...
  }
}
```

**步骤 4: 创建回顾文档（可选）**
如果用户同意：
```bash
cp templates/retrospective-template.md \
   execution/retrospectives/retro-项目名-20260304.md
```

填写回顾内容：
- 做得好的方面
- 需要改进的地方
- 关键数据指标
- 经验教训

**步骤 5: 更新关联文档**
如果项目有路线图、提案等，添加归档说明

**步骤 6: 生成项目总结（可选）**
从项目文档中提取：
- 项目目标达成情况
- 关键里程碑完成时间
- 主要成果
- 团队成员贡献

**步骤 7: 确认归档**
向用户展示：
- 已归档的文档位置
- 注册表状态更新
- 是否创建了回顾文档

### 示例对话

```
用户：域名规划项目完成了

AI：恭喜！🎉 我来帮你归档项目。

确认信息：
- 项目状态：已完成
- 是否创建回顾文档？（建议创建，总结经验教训）

用户：是的，创建回顾文档

AI：好的，正在处理...

✅ 归档完成

📂 已移动文档：
- projects/active/域名规划与基础设施-20260304.md
  → projects/archived/域名规划与基础设施-20260304-completed.md

📋 已更新注册表：
- 状态：active → completed
- 更新日期：2026-06-30

📝 已创建回顾文档：
- execution/retrospectives/retro-域名规划-20260630.md

📊 项目总结
- 项目周期：2026-03-04 至 2026-06-30（118天）
- 里程碑：6个全部完成
- 成果：55个子域名规划，10个核心域名上线

建议：
在回顾文档中记录本项目的经验教训，供后续项目参考。

是否需要我帮你填写回顾文档？
```

## 工作流 5: 生成报告

### 触发条件
用户说："生成项目报告"、"这周做了什么"、"总结一下进展"

### 操作步骤

**步骤 1: 确定报告类型**
- 周报/月报（定期报告）
- 项目总结报告
- 里程碑报告

**步骤 2: 收集数据**
```bash
# 读取所有活跃项目
find projects/registry -name "*.json" -exec jq 'select(.status == "active")' {} \;

# 提取关键信息
- 本周完成的里程碑
- 正在进行的任务
- 遇到的问题
- 下周计划
```

**步骤 3: 创建报告文档**
```bash
cp templates/report-template.md execution/reports/report-2026-W10.md
```

**步骤 4: 填写报告内容**
```markdown
## 本周完成工作
- ✅ 项目A：完成里程碑X
- ✅ 项目B：完成任务Y

## 进行中的工作
- 🟢 项目C：开发功能Z（进度 60%）

## 风险与问题
- ⚠️ 项目D：资源不足，可能延期

## 下周计划
- 项目A：开始下一阶段
- 项目C：完成功能Z开发
```

**步骤 5: 添加数据可视化（可选）**
- 项目数量统计
- 完成率
- 进度图表

**步骤 6: 输出报告**
提供 Markdown 格式或建议导出为 PDF

### 示例对话

```
用户：生成本周的工作报告

AI：好的，我来汇总本周（2026-W10）的工作进展。

正在分析所有活跃项目...

📊 本周工作报告（2026-03-04 至 2026-03-10）

✅ 本周完成
- 域名规划项目：Phase 1 子域名注册完成
- 完成 2 个里程碑

🟢 进行中
- 域名规划项目：官网开发（进度 30%）
- 1 个项目正常推进

⚠️ 风险与问题
- 暂无阻塞问题

📅 下周计划
- 完成官网首页设计
- 开始文档站开发

📈 关键指标
- 活跃项目：1 个
- 本周完成率：33%（2/6 里程碑）
- 整体进度：30%

已保存报告至：execution/reports/report-2026-W10.md

是否需要我添加更详细的内容？
```

## 🎯 工作流执行检查清单

每次执行工作流后，AI 应检查：

### 通用检查
- [ ] 文件命名符合规范
- [ ] 日期格式一致（YYYY-MM-DD）
- [ ] 文档格式正确（Markdown/JSON）
- [ ] 相对路径正确

### Markdown 文档
- [ ] 包含元信息（标题、日期、负责人）
- [ ] 标题层级正确
- [ ] 表格格式完整
- [ ] 任务列表格式正确

### JSON 注册表
- [ ] JSON 格式正确（可解析）
- [ ] 必填字段完整
- [ ] 文档路径与实际文件一致
- [ ] `updated_date` 已更新

### 一致性检查
- [ ] Markdown 和 JSON 状态一致
- [ ] 项目名称在所有文档中一致
- [ ] 里程碑信息同步
