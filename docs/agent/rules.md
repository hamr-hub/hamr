# AI 助手行为准则

AI 助手在 HamR 项目知识库中的行为规范和约束。

## 🎯 核心原则

### 1. 一致性优先
**原则**：保持 Markdown 文档和 JSON 注册表的信息同步

**DO ✅**
- 创建项目文档时，同步创建注册表
- 更新文档时，同步更新注册表的 `updated_date`
- 修改项目状态时，同时更新 Markdown 和 JSON

**DON'T ❌**
- 只创建 Markdown 不创建 JSON
- 修改文档后忘记更新 `updated_date`
- Markdown 和 JSON 中的状态不一致

**示例**
```
❌ 错误做法
- 创建了 projects/active/新项目.md
- 但没有创建 projects/registry/PROJ-XXX.json

✅ 正确做法
- 创建 projects/active/新项目-20260304.md
- 同时创建 projects/registry/PROJ-002-新项目.json
- 在 JSON 中记录文档路径
```

---

### 2. 结构化思考
**原则**：先读取注册表定位项目，再读取具体文档获取详情

**DO ✅**
- 用户询问项目信息时，先读取注册表
- 从注册表获取文档路径，再读取具体文档
- 使用注册表快速筛选和查找项目

**DON'T ❌**
- 跳过注册表，直接搜索文件名
- 不使用注册表的文档路径信息
- 假设文档位置，不验证路径

**查询顺序**
```
第1步：读取注册表
read projects/registry/PROJ-001-域名规划.json

第2步：获取文档路径
jq -r '.documents.project' PROJ-001-域名规划.json
→ projects/active/域名规划与基础设施-20260304.md

第3步：读取具体文档
read projects/active/域名规划与基础设施-20260304.md

第4步：提取信息并回复用户
```

---

### 3. 遵循规范
**原则**：严格遵循文档命名和格式规范

**DO ✅**
- 使用标准模板创建文档
- 文件命名：`项目名称-YYYYMMDD.md`
- 日期格式：`YYYY-MM-DD`
- 项目 ID 格式：`PROJ-NNN`（三位数字）

**DON'T ❌**
- 使用非标准命名（如 `项目.md`、`20260304-项目.md`）
- 使用非标准日期格式（如 `2026/03/04`、`03-04-2026`）
- 跳过模板，从零开始写文档

**命名规范**
```
✅ 正确
- projects/active/用户认证系统-20260304.md
- projects/registry/PROJ-002-用户认证.json
- planning/roadmaps/用户认证-roadmap-2026Q2.md

❌ 错误
- projects/active/用户认证.md（缺少日期）
- projects/registry/用户认证.json（缺少 ID）
- planning/roadmaps/roadmap-2026Q2.md（缺少项目名）
```

---

### 4. 主动提示
**原则**：主动发现问题并提醒用户

**DO ✅**
- 项目缺少必要文档时提醒
- 里程碑临近时提醒（提前3天）
- 项目状态需要更新时建议
- 文档不一致时告警

**DON'T ❌**
- 发现问题不告诉用户
- 被动等待用户询问
- 假设用户知道所有规范

**提示示例**
```
场景1：缺少路线图
用户：创建了新项目
AI：✅ 项目文档已创建
    💡 建议：是否同时创建产品路线图？这有助于规划实施阶段。

场景2：里程碑临近
AI：⏰ 提醒：官网上线的里程碑将在 3 天后到期（2026-04-15）
    当前进度：30%
    建议：加快开发进度或调整里程碑日期

场景3：状态不一致
AI：⚠️ 发现问题：
    - Markdown 中状态为"进行中"
    - JSON 中状态为"已完成"
    是否需要我修正这个不一致？
```

---

### 5. 用户确认
**原则**：重要操作前必须征得用户同意

**DO ✅**
- 归档项目前确认
- 删除文档前确认
- 修改项目状态前确认
- 批量操作前确认

**DON'T ❌**
- 自动归档项目
- 自动删除文档
- 假设用户意图

**确认示例**
```
❌ 错误做法
用户：这个项目做完了
AI：[自动归档项目]
    ✅ 项目已归档

✅ 正确做法
用户：这个项目做完了
AI：确认要归档项目吗？
    
    操作：
    - 移动文档到 archived/ 目录
    - 更新状态为 "completed"
    
    是否继续？（是/否）
```

## 📋 操作规范

### 创建文档

**步骤检查清单**
- [ ] 使用标准模板
- [ ] 文件命名符合规范
- [ ] 填写所有必填字段
- [ ] 同步创建注册表（如果是项目文档）
- [ ] 验证文档格式正确
- [ ] 向用户确认创建成功

**质量标准**
- 文件名：`项目名称-YYYYMMDD.md`
- 元信息：包含标题、日期、负责人、状态
- 内容：按模板结构填写，不跳过章节
- 格式：表格完整、列表正确、链接有效

---

### 更新文档

**步骤检查清单**
- [ ] 读取当前文档内容
- [ ] 确认要更新的信息
- [ ] 修改文档内容
- [ ] 同步更新注册表（如果需要）
- [ ] 更新 `updated_date` 或 `最后更新` 字段
- [ ] 验证更新后的一致性
- [ ] 向用户确认更新成功

**一致性检查**
- Markdown 和 JSON 的状态一致
- 里程碑信息同步
- 日期格式一致
- 项目名称一致

---

### 查询信息

**查询优先级**
1. **优先级 1**：读取注册表（JSON）
   - 用于快速获取项目摘要
   - 定位文档路径
   
2. **优先级 2**：读取项目主文档
   - 用于获取详细信息
   - 查看完整的项目计划

3. **优先级 3**：读取关联文档
   - 路线图、提案、回顾等
   - 根据用户需求选择性读取

**回复格式**
```
📊 项目信息（简要）
- 基本信息（ID、名称、状态）
- 关键里程碑
- 文档路径

如需详细信息，询问用户：
"是否需要查看详细内容？"
```

---

### 归档项目

**步骤检查清单**
- [ ] 确认归档原因（完成/取消）
- [ ] 移动项目文档到 archived/ 目录
- [ ] 添加状态后缀（-completed 或 -cancelled）
- [ ] 更新注册表状态
- [ ] 更新文档路径
- [ ] 询问是否创建回顾文档
- [ ] 生成项目总结（可选）
- [ ] 向用户确认归档完成

**状态后缀**
- `-completed`：项目成功完成
- `-cancelled`：项目取消
- `-paused`：项目暂停（仍在 active/ 目录）

## 🚫 禁止行为

### 1. 破坏格式
**禁止**：
- 使用非标准日期格式
- 破坏表格结构
- 删除必填字段
- 混用不同的命名规范

**后果**：
- 文档难以解析
- 自动化工具失效
- 团队协作困难

---

### 2. 跳过注册表
**禁止**：
- 创建项目文档不创建注册表
- 更新文档不更新注册表
- 查询项目不读取注册表

**后果**：
- 项目信息散乱
- 无法快速检索
- 文档完整性无法追踪

---

### 3. 假设信息
**禁止**：
- 没有读取文档就回答问题
- 假设文档位置
- 猜测项目状态

**后果**：
- 提供错误信息
- 误导用户决策
- 降低用户信任

---

### 4. 自作主张
**禁止**：
- 未经确认归档/删除项目
- 自动修改重要信息
- 批量操作不征求同意

**后果**：
- 误操作难以恢复
- 破坏用户数据
- 引起用户不满

---

### 5. 忽略规范
**禁止**：
- 不使用模板
- 自创文档结构
- 违反命名约定

**后果**：
- 文档风格不一致
- 团队理解成本增加
- 维护困难

---

## ✅ 推荐行为

### 1. 主动检查
定期检查：
- 注册表和文档是否一致
- 即将到期的里程碑
- 长时间未更新的项目
- 缺失的必要文档

---

### 2. 智能建议
根据上下文提供建议：
- 项目创建后建议添加里程碑
- 完成阶段后建议创建回顾
- 状态变更后建议更新路线图

---

### 3. 友好沟通
与用户交互时：
- 使用清晰的表达
- 提供具体的操作步骤
- 解释操作的原因和影响
- 使用 emoji 增强可读性（适度）

---

### 4. 持续学习
从每次操作中学习：
- 记录常见问题
- 总结最佳实践
- 优化工作流程

---

- [ ] 格式符合模板
- [ ] 必填字段完整
- [ ] 日期格式一致
- [ ] 链接路径正确

### 一致性
- [ ] Markdown 和 JSON 状态一致
- [ ] 文档路径与实际文件一致
- [ ] 项目信息在所有文档中一致
- [ ] 里程碑信息同步

### 用户体验
- [ ] 操作结果清晰反馈
- [ ] 提供下一步建议
- [ ] 重要操作已确认
- [ ] 问题主动提示
