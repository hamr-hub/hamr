## 🎯 项目概述

### 项目背景
HamR 管家是家庭本地化数据管理应用，采用 **P2P 架构**和 **Web3 技术**，确保用户数据完全私有化。面向 3-5 人家庭场景，通过局域网或本地设备部署，实现去中心化的五维管理（人、时、事、物、境），核心是**隐私保护**与**数据主权**。

### 项目目标
- 目标1：实现五维管理基础功能（人员、时间、事务、物品、环境）
- 目标2：P2P 本地化部署，支持局域网内多设备同步
- 目标3：Web3 身份认证，保护用户隐私
- 目标4：端到端加密，数据完全私有化
- 目标5：纯 C2C 应用，无需云服务器

### 项目价值
- **用户价值**：数据完全自主可控，隐私保护，无第三方存储
- **业务价值**：低成本（无服务器成本），高信任度（数据私有）
- **技术价值**：P2P 架构实践，去中心化数据同步，Web3 身份管理

## 📐 项目范围

### 核心职责
HamR 管家是**P2P 本地化家庭管理系统**，负责：
1. **五维数据本地存储**：数据存储在用户设备，不上传云端
2. **局域网多设备同步**：家庭内 3-5 台设备 P2P 同步
3. **Web3 身份认证**：去中心化身份（DID），无需账号系统
4. **端到端加密通信**：设备间数据传输加密
5. **数据完全私有化**：用户拥有数据绝对控制权

### 部署模式

#### 模式1: 本地 App（桌面/移动端）
- 单设备运行，数据本地存储
- 适合个人使用或单设备家庭

#### 模式2: 家庭局域网部署
- 一台设备作为主节点（如树莓派/NAS）
- 其他设备作为客户端连接
- 局域网内 P2P 同步

#### 模式3: 本地服务器部署
- 支持在家庭服务器/NAS 上部署
- 提供 Web 界面访问
- 内网穿透可选（如需远程访问）

## 🛠 技术方案

### 技术栈
- **前端**：React + TypeScript + Tailwind CSS（桌面端 Electron，移动端 React Native）
- **后端**：Rust + Axum（本地服务）
- **数据库**：SQLite（本地嵌入式数据库）
- **P2P 同步**：libp2p / IPFS
- **身份认证**：Web3 DID（去中心化身份）
- **加密**：X25519（密钥交换）+ ChaCha20-Poly1305（数据加密）
- **网络**：mDNS（局域网发现）+ QUIC（传输层）

### P2P 同步机制

#### 1. 设备发现
- **mDNS（Multicast DNS）**：局域网内自动发现其他设备
- **设备标识**：每台设备生成唯一 DID（去中心化身份）
- **信任机制**：首次连接需扫码/密钥确认

#### 2. 数据同步
- **CRDT（Conflict-free Replicated Data Types）**：无冲突复制数据类型
- **增量同步**：仅同步变更部分，减少网络开销
- **最终一致性**：所有设备最终达到一致状态

#### 3. 加密通信
- **密钥交换**：X25519 椭圆曲线密钥交换
- **数据传输**：ChaCha20-Poly1305 加密
- **端到端加密**：数据在设备间传输全程加密

### Web3 身份认证
```
【去中心化身份（DID）流程】
1. 首次使用：生成密钥对（Ed25519）
2. 创建 DID：did:key:z6MkhaXgBZDvotDkL5257faiztiGiC2QtKLGpbnnEGta2doK
3. 家庭成员邀请：扫码/分享公钥
4. 权限管理：基于签名验证访问权限
5. 无需账号系统：密钥即身份
```

### [2026-03-18] 架构调整：从云端 SaaS 到 P2P 本地化

  2. **降低成本**：无需云服务器，年度节省 ¥31,200

  4. **Web3 特性**：去中心化身份，无需账号系统
- **影响范围**：架构重构、技术栈调整、开发计划重新制定
- **状态**：🔄 进行中

- **背景**：P2P 本地化部署需要嵌入式数据库
