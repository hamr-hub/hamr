## 🎯 项目概述

### 项目背景
HamR 项目包含两大系统（HamR管家 + JiaBu决策助手），需要建立清晰的域名体系以支撑：
1. **C端用户服务**：家庭用户使用产品的官方门户
2. **开发者生态**：技术文档、开源协作、私有部署的专业平台

已注册域名：
- `hamr.store` - 拟作为家庭用户服务主域名
- `hamr.top` - 拟作为开发者生态域名

### 项目目标
- **目标1**：建立清晰的双域名定位和边界，避免用户混淆
- **目标2**：规划55个子域名的功能分配和实施优先级
- **目标3**：完成Phase 1核心子域名（10个）的部署上线
- **目标4**：制定域名安全、监控、应急预案的完整方案

### 项目价值
- **用户价值**：清晰的产品入口，专业的品牌形象，稳定的服务体验
- **业务价值**：支撑产品增长，降低品牌混淆风险，为国际化预留空间
- **技术价值**：架构清晰，开发者友好，利于开源生态建设

## 🛠 技术方案

### 技术栈
- **DNS管理**：Cloudflare（免费SSL、CDN、DDoS防护）
- **官网(hamr.store)**：静态站点生成器（Next.js/Hugo）
- **文档站(hamr.top)**：VitePress/Docusaurus
- **CDN**：Cloudflare（全球加速）
- **监控**：UptimeRobot（域名可用性）+ Sentry（错误追踪）

### 架构设计

#### hamr.store 架构
```
用户 → Cloudflare CDN → Web应用服务器
                        ├─ 静态官网（Nginx）
                        ├─ app.hamr.store（应用服务）
                        └─ account.hamr.store（账号服务）
                        ↓
                     数据库集群（加密存储）
```

#### hamr.top 架构
```
开发者 → Cloudflare CDN → 文档/工具服务器
                         ├─ docs.hamr.top（GitHub Pages）
                         ├─ api.hamr.top（API网关）
                         └─ demo.hamr.top（演示环境）
```

### 技术依赖
- Cloudflare DNS：域名解析和CDN服务
- GitHub Pages：文档站托管（备选方案）
- SSL证书：Let's Encrypt 自动续期

## 📝 关键决策记录

### [2026-03-04] 双域名定位确认
- **背景**：需要明确两个域名的边界，避免功能重叠
- **决策内容**：
  - hamr.store = 家庭用户服务主域名（温暖、可信赖）
  - hamr.top = 开发者生态域名（技术、开放、极客）
- **决策原因**：
  - .store 后缀符合"家庭商店/管家"的产品定位
  - .top 后缀符合技术品牌的专业形象
  - 双域名分离避免用户与开发者流量混淆
- **影响范围**：所有后续子域名分配、品牌宣传、SEO策略

### [2026-03-04] 子域名数量确认（55个）
- **背景**：需要平衡功能完整性与管理复杂度
- **决策内容**：
  - hamr.store：30个子域名（核心产品+用户服务+社区）
  - hamr.top：25个子域名（文档+工具+开源协作）
- **决策原因**：
  - 覆盖产品生命周期（规划、开发、运营、增长）
  - 为未来3年发展预留空间
  - 子域名管理成本可控（DNS记录管理）
- **影响范围**：运维工作量、监控点数量、SSL证书管理

### [2026-03-04] Phase 1 优先级确认
- **背景**：资源有限，需分阶段实施
- **决策内容**：Phase 1（Q2）仅实施10个核心子域名：
  - hamr.store（官网）
  - app.hamr.store（HamR应用）
  - account.hamr.store（账号中心）
  - help.hamr.store（帮助中心）
  - hamr.top（开发者门户）
  - docs.hamr.top（技术文档）
  - api.hamr.top（API服务）
  - deploy.hamr.top（部署指南）
  - demo.hamr.top（在线演示）
  - status.hamr.top（服务状态）
- **决策原因**：
  - 满足产品MVP上线的最小需求
  - 控制初期成本和复杂度
  - 快速验证双域名体系的可行性
- **影响范围**：开发排期、预算分配、资源投入

### [2026-03-04] 数据隔离策略确认
- **背景**：家庭隐私数据极为敏感，需要严格隔离
- **决策内容**：
  - hamr.store：存储加密的家庭数据，不与hamr.top共享
  - hamr.top：仅存储公开技术文档和匿名化的API日志
  - 双域名使用独立的数据库集群
- **决策原因**：
  - 符合隐私优先的产品理念
  - 降低数据泄露风险
  - 利于通过安全审计和合规认证
- **影响范围**：数据库架构、成本增加、备份策略
