## 🎯 项目概述

### 项目背景

**核心理念**：
> "家家有本难念的经" —— 我们致力于打造免费的家庭智能助理，帮助每个家庭更好地发展和管理，成就人类家庭的命运共同体。

**产品定位**：
HamR 是一个由两大分布式系统组成的家庭智能助理平台：

1. **哈默尔（HamR）** - 家庭大管家
   - 存储并管理家中各种人、事、物数据
   - 辅助收集家族信息（支持私有部署，不涉及隐私）
   - 评估家庭幸福感
   - 保存大事记，让美好一直存在

2. **家布（JiaBu）** - 智能决策助手
   - 辅助决策，提升家庭幸福感
   - 智能提醒，让家中惊喜连连
   - 模拟事物发展，谋划布局未来
   - 监测环境，给出行动优化建议

**五维管理体系**：从 **人、时、事、物、境** 五个维度综合管理家庭事务

**域名体系需求**：
为支撑这一愿景，需要建立清晰的双域名体系：
1. **C端用户服务**：温暖、可信赖的家庭用户服务门户
2. **开发者生态**：开放、专业的技术协作平台

已注册域名：
- `hamr.store` - 家庭用户服务主域名（"家庭商店/管家"的温暖定位）
- `hamr.top` - 开发者生态域名（技术领先、开放协作的专业形象）

### 项目目标
- **目标1**：建立清晰的双域名定位和边界，体现"温暖家庭"与"专业技术"的双重品牌形象
- **目标2**：规划55个子域名的功能分配，覆盖"人、时、事、物、境"五维管理体系
- **目标3**：完成Phase 1核心子域名（10个）的部署上线，支撑产品MVP
- **目标4**：制定隐私优先的安全方案，确保家庭数据绝对安全
- **目标5**：为免费产品和私有部署提供清晰的技术路径

### 项目价值
- **用户价值**：
  - 温暖可信的产品入口，让家庭用户感到安心
  - 隐私优先的数据保护，支持私有部署
  - 免费使用，降低家庭数字化门槛

- **社会价值**：
  - 帮助家庭建立幸福管理体系
  - 促进家庭成员沟通与协作
  - 成就人类家庭的命运共同体

- **技术价值**：
  - 开放的开发者生态，鼓励社区贡献
  - 清晰的技术架构，利于私有部署
  - 为开源协作提供专业平台

#### hamr.store 架构（隐私优先）
```
家庭用户 → 阿里云 CDN → Web应用服务器
                             ├─ 静态官网（Nginx）
                             ├─ app.hamr.store（HamR管家服务）
                             ├─ jiabu.hamr.store（JiaBu决策服务）
                             ├─ account.hamr.store（账号服务 - SSO）
                             └─ family.hamr.store（家庭管理）
                             ↓
                          数据库集群（AES-256加密存储）
                          ├─ 家庭数据：完全隔离，不跨域共享
                          ├─ 用户数据：端到端加密
                          └─ 备份：本地加密 + 可选云备份
```

#### hamr.top 架构（开放协作）
```
开发者 → 阿里云 CDN → 文档/工具服务器
                          ├─ docs.hamr.top（GitHub Pages / VitePress）
                          ├─ api.hamr.top（API网关 - 公开接口）
                          ├─ demo.hamr.top（演示环境 - 脱敏数据）
                          └─ deploy.hamr.top（私有部署指南）
                          ↓
                       技术内容存储
                       ├─ 公开文档：开源仓库同步
                       ├─ API日志：仅匿名化数据
                       └─ 无家庭隐私数据
```

#### 私有部署方案
```
用户自有服务器 → 用户自有域名（或 *.hamr.store 子域名）
                 ├─ Docker Compose 一键部署
                 ├─ 数据完全本地化
                 ├─ 可选：接入 HamR 更新服务
                 └─ 可选：接入 JiaBu 智能决策（匿名化数据交换）
```

## 📝 关键决策记录

### [2026-03-04] 双域名定位确认
- **背景**：HamR 包含面向家庭用户的管家服务和面向开发者的开源生态，需明确边界
- **决策内容**：
  - hamr.store = 家庭用户服务主域名（温暖、可信赖、像"家"一样的感觉）
  - hamr.top = 开发者生态域名（技术、开放、极客精神）
- **决策原因**：
  - .store 后缀符合"家庭管家/智能商店"的产品定位，传递温暖感
  - .top 后缀符合技术品牌的专业形象，强调技术领先
  - 双域名分离实现"家庭隐私"与"开放协作"的天然隔离
  - 家庭用户不会误入技术区域，开发者有独立的协作空间
- **影响范围**：所有后续子域名分配、品牌宣传、SEO策略

### [2026-03-04] 子域名数量确认（55个）
- **背景**：需要覆盖两大系统的完整功能，同时控制管理复杂度
- **决策内容**：
  - hamr.store：30个子域名（HamR管家 + JiaBu决策 + 用户服务 + 社区）
  - hamr.top：25个子域名（文档 + 工具 + 开源协作 + 部署方案）
- **子域名与五维管理体系的映射**：
  - **人**：account / family / profile / invite（人员与家庭关系管理）
  - **时**：calendar / reminder / timeline（时间规划与大事记）
  - **事**：app / jiabu / data / report（事务管理与决策）
  - **物**：album / archive / shop（物品记录与存储）
  - **境**：settings / notification / events（环境监测与场景联动）
- **决策原因**：
  - 覆盖产品从管家到决策的全功能链
  - 为"人时事物境"五维管理预留充足空间
  - 子域名管理成本可控（DNS记录管理）
- **影响范围**：运维工作量、监控点数量、SSL证书管理

### [2026-03-04] Phase 1 优先级确认
- **背景**：资源有限，需分阶段实施，优先验证核心价值
- **决策内容**：Phase 1（Q2）仅实施10个核心子域名：
  - **hamr.store（4个）**：
    - hamr.store（官网 - 产品理念展示）
    - app.hamr.store（HamR管家应用 - 核心价值验证）
    - account.hamr.store（账号中心 - 家庭身份管理）
    - help.hamr.store（帮助中心 - 降低使用门槛）
  - **hamr.top（6个）**：
    - hamr.top（开发者门户 - 开源社区入口）
    - docs.hamr.top（技术文档 - 开发者友好）
    - api.hamr.top（API服务 - 功能开放）
    - deploy.hamr.top（部署指南 - 私有部署支持）
    - demo.hamr.top（在线演示 - 快速体验）
    - status.hamr.top（服务状态 - 透明可信）
- **决策原因**：
  - 满足"家庭管家 + 私有部署"的最小可行产品
  - 快速验证免费家庭智能助理的核心价值
  - 控制初期成本，适合小团队启动
- **影响范围**：开发排期、预算分配、资源投入

### [2026-03-04] 数据隔离策略确认
- **背景**：家庭隐私数据极为敏感，"可私有化部署、不涉及家庭隐私"是核心承诺
- **决策内容**：
  - hamr.store：存储加密的家庭数据，不与hamr.top共享任何用户数据
  - hamr.top：仅存储公开技术文档和匿名化的API日志
  - 双域名使用独立的数据库集群，物理隔离
  - 私有部署方案：数据完全留在用户自有服务器
- **决策原因**：
  - 兑现"不涉及家庭隐私"的产品承诺
  - 降低数据泄露风险，一个域名被攻击不影响另一个
  - 利于通过安全审计和合规认证（GDPR、个人信息保护法）
  - 为私有部署用户提供完全的数据自主权
- **影响范围**：数据库架构、成本增加、备份策略、私有部署方案

### [2026-03-05] 五维管理体系与域名映射
- **背景**：JiaBu 系统从"人、时、事、物、境"五个维度综合管理家庭事务，域名体系需反映这一设计哲学
- **决策内容**：
  - 子域名命名优先体现功能语义，便于家庭用户直觉理解
  - 每个维度至少有2个对应子域名，确保功能完整性
  - 五维管理的入口统一收敛至 `app.hamr.store`（HamR管家）和 `jiabu.hamr.store`（JiaBu决策）
- **决策原因**：
  - 家庭用户无需理解五维模型，通过自然的功能名称进入
  - 开发者通过文档站了解五维架构，便于API设计和扩展
  - 保持域名简洁直观，降低认知成本
- **影响范围**：子域名命名、产品导航设计、API分组

### [2026-03-06] DNS 更新操作规范
- **工具**：`repos/hamr-infra/scripts/dns/`

  4. 执行 `./update-dns.sh` 正式更新
