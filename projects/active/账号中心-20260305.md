## 🎯 项目概述

### 项目背景
HamR 平台需要统一的身份认证和家庭管理系统，支持用户注册、登录、家庭创建、成员邀请等核心功能。账号中心是整个平台的身份基础设施，需要确保安全性和易用性。

### 项目目标
- 目标1：实现用户注册、登录、密码找回功能
- 目标2：支持家庭创建和成员管理（邀请、角色权限）
- 目标3：提供 SSO 单点登录，支持全平台域名（*.hamr.store）
- 目标4：确保数据安全（AES-256加密、端到端加密）
- 目标5：支持私有部署的本地认证方案

### 项目价值
- **用户价值**：便捷安全的账号体系，一次登录全平台可用
- **业务价值**：用户身份管理，支撑所有业务功能
- **技术价值**：统一认证中心，降低各子系统开发成本

## 📐 项目范围

### 核心职责
账号中心是 HamR 平台的**身份认证与授权中心**，负责：
1. **用户身份管理**：注册、登录、密码管理
2. **家庭组织管理**：家庭创建、成员邀请、角色权限
3. **统一认证服务**：为所有子域名提供 SSO 单点登录
4. **会话管理**：JWT 签发、刷新、撤销
5. **安全审计**：登录历史、异常检测、安全日志

### 包含的功能

#### 1. 用户管理模块
- [ ] **用户注册**
  - 邮箱注册（验证码验证）
  - 手机号注册（短信验证）
  - 用户名唯一性检查
  - 密码强度验证（至少 8 位，包含大小写字母、数字）
- [ ] **用户登录**
  - 密码登录
  - 验证码登录（邮箱/手机）
  - 记住登录状态（7天/30天）
  - 多设备登录管理
- [ ] **密码管理**
  - 密码找回（邮箱/手机验证）
  - 密码重置
  - 密码修改（需验证旧密码）
  - 密码加密存储（bcrypt）

#### 2. 家庭管理模块
- [ ] **家庭创建**
  - 创建家庭（自动成为管理员）
  - 家庭名称、头像、简介
  - 家庭唯一标识符（Family ID）
- [ ] **成员邀请**
  - 生成邀请码（6位数字，24小时有效）
  - 生成邀请链接（带 Token）
  - 邀请记录追踪
  - 邀请接受/拒绝
- [ ] **成员管理**
  - 成员列表查看
  - 成员角色分配（管理员/成员/观察者）
  - 成员移除（仅管理员）
  - 成员权限控制
- [ ] **角色权限定义**
  - **管理员**：完全控制权（邀请、移除、修改家庭信息）
  - **成员**：读写权限（查看和编辑家庭数据）
  - **观察者**：只读权限（仅查看数据）

#### 3. SSO 单点登录模块
- [ ] **OAuth2 认证流程**
  - 授权码模式（Authorization Code）
  - 客户端凭证模式（Client Credentials）
  - 刷新令牌机制（Refresh Token）
- [ ] **JWT 令牌管理**
  - Access Token 签发（15分钟有效期）
  - Refresh Token 签发（7天有效期）
  - Token 刷新接口
  - Token 撤销（登出）
- [ ] **跨域认证**
  - 支持 *.hamr.store 和 *.hamr.top 全域名
  - Cookie 共享（SameSite=Lax）
  - CORS 配置
- [ ] **认证回调**
  - 登录成功回调（redirect_uri）
  - 登出回调
  - 会话过期处理

#### 4. 用户资料模块
- [ ] **基本信息**
  - 昵称、头像、性别、生日
  - 邮箱、手机号（需验证）
  - 个人简介
- [ ] **隐私设置**
  - 资料可见性（公开/家庭内/仅自己）
  - 数据导出请求
  - 账号注销申请

#### 5. 账号安全模块
- [ ] **二步验证（2FA）**
  - TOTP 验证器（Google Authenticator）
  - 备用验证码（10个）
  - 验证器绑定/解绑
- [ ] **登录历史**
  - 登录时间、IP、设备、位置
  - 异常登录检测（新设备/新地点）
  - 登录通知（邮件/短信）
- [ ] **安全日志**
  - 密码修改记录
  - 权限变更记录
  - 敏感操作审计

#### 6. API 接口（供其他服务调用）
- [ ] **认证接口**
  - `POST /api/auth/register` - 用户注册
  - `POST /api/auth/login` - 用户登录
  - `POST /api/auth/logout` - 用户登出
  - `POST /api/auth/refresh` - 刷新 Token
  - `GET /api/auth/verify` - 验证 Token 有效性
- [ ] **用户接口**
  - `GET /api/users/me` - 获取当前用户信息
  - `PUT /api/users/me` - 更新用户信息
  - `GET /api/users/:id` - 获取指定用户信息（需权限）
- [ ] **家庭接口**
  - `POST /api/families` - 创建家庭
  - `GET /api/families/:id` - 获取家庭信息
  - `PUT /api/families/:id` - 更新家庭信息
  - `POST /api/families/:id/invite` - 邀请成员
  - `GET /api/families/:id/members` - 获取成员列表
  - `DELETE /api/families/:id/members/:userId` - 移除成员

## 🛠 技术方案

### 技术栈
- **后端框架**：Node.js + Express / Go + Gin
- **数据库**：PostgreSQL（用户数据）+ Redis（Session）
- **认证协议**：OAuth2 + JWT
- **加密**：bcrypt（密码）+ AES-256（敏感数据）
- **前端**：React + Ant Design
- **域名**：account.hamr.store

【SSO 认证流程】
其他子域名（如 app.hamr.store）
    ↓
1. 检查本地 JWT Token
    ↓ (无效或过期)
2. 重定向到 account.hamr.store/login?redirect_uri=xxx
    ↓
3. 用户登录成功
    ↓
4. 签发 JWT Token（Access + Refresh）
    ↓
5. 重定向回原服务（带 Token）
    ↓
6. 原服务验证 Token（调用 /api/auth/verify）
    ↓

    ↓

### 数据库设计（核心表结构）

#### 用户表（users）
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(255) UNIQUE NOT NULL,
    phone VARCHAR(20) UNIQUE,
    password_hash VARCHAR(255) NOT NULL,  -- bcrypt
    nickname VARCHAR(100),
    avatar_url TEXT,
    gender VARCHAR(10),
    birthday DATE,
    bio TEXT,
    email_verified BOOLEAN DEFAULT FALSE,
    phone_verified BOOLEAN DEFAULT FALSE,
    two_factor_enabled BOOLEAN DEFAULT FALSE,
    two_factor_secret VARCHAR(255),  -- AES-256 加密
    status VARCHAR(20) DEFAULT 'active',  -- active/suspended/deleted
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW(),
    last_login_at TIMESTAMP
);
```

#### 家庭表（families）
```sql
CREATE TABLE families (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    avatar_url TEXT,
    description TEXT,
    owner_id UUID NOT NULL REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

#### 家庭成员表（family_members）
```sql
CREATE TABLE family_members (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    family_id UUID NOT NULL REFERENCES families(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    role VARCHAR(20) NOT NULL,  -- admin/member/observer
    joined_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(family_id, user_id)
);
```

#### 邀请表（invitations）
```sql
CREATE TABLE invitations (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    family_id UUID NOT NULL REFERENCES families(id) ON DELETE CASCADE,
    inviter_id UUID NOT NULL REFERENCES users(id),
    invite_code VARCHAR(6) UNIQUE,  -- 6位数字
    invite_token VARCHAR(255) UNIQUE,  -- 链接 Token
    email VARCHAR(255),
    phone VARCHAR(20),
    status VARCHAR(20) DEFAULT 'pending',  -- pending/accepted/rejected/expired
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT NOW(),
    accepted_at TIMESTAMP
);
```

#### 登录历史表（login_history）
```sql
CREATE TABLE login_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    ip_address INET NOT NULL,
    user_agent TEXT,
    device_type VARCHAR(50),
    location VARCHAR(255),
    status VARCHAR(20) NOT NULL,  -- success/failed
    failure_reason TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 📝 关键决策记录

### [2026-03-05] 认证方案选择
- **背景**：需要统一认证，支持 SSO
- **决策内容**：采用 OAuth2 + JWT 方案
- **决策原因**：标准协议、跨域友好、无状态认证
- **影响范围**：所有子域名的认证逻辑

### [2026-03-05] 数据加密策略
- **背景**：家庭数据极为敏感，需确保隐私
- **决策内容**：密码用 bcrypt、敏感字段用 AES-256、数据库整体加密
- **决策原因**：行业标准、符合隐私承诺
- **影响范围**：数据库设计、性能
