## 🎯 项目概述

### 项目背景
为开发者提供统一的 API 网关，支持第三方应用接入、数据查询、智能决策调用等功能，是开放生态的技术基础。

### 项目目标
- 目标1：实现 RESTful API 网关
- 目标2：提供 API 鉴权和限流机制
- 目标3：支持 API 版本管理
- 目标4：实现请求日志和监控
- 目标5：提供 Webhook 回调机制

### 项目价值
- **用户价值**：第三方应用可调用 HamR 数据和服务
- **业务价值**：开放生态、扩展能力
- **技术价值**：标准化 API 设计、微服务架构

## 📐 项目范围

### 核心职责
API 网关是 HamR 平台的**统一 API 入口**，负责：
1. **统一入口管理**：所有外部 API 请求经由网关分发到后端微服务
2. **安全防护**：API 鉴权、限流、防滥用
3. **流量治理**：请求路由、负载均衡、熔断降级
4. **可观测性**：请求日志、性能监控、调用链追踪
5. **开发者支持**：API 密钥管理、文档生成、SDK 支持

### 包含的功能

#### 1. API 网关核心
- [ ] **请求路由**
  - URL 路径匹配路由到对应微服务
  - 路由表配置（动态加载）
  - 路由优先级排序
  - 路径重写（path rewrite）
- [ ] **负载均衡**
  - 轮询策略（Round Robin）
  - 加权策略（Weighted）
  - 最少连接策略（Least Connections）
  - 健康检查（主动/被动）
- [ ] **协议处理**
  - HTTP/HTTPS 支持
  - 请求/响应头管理
  - 请求体大小限制（默认 10MB）
  - 超时配置（连接超时、读取超时）

#### 2. 安全与鉴权
- [ ] **API Key 认证**
  - API Key 生成与管理
  - Key 分级（开发/生产）
  - Key 使用统计
  - Key 吊销与轮换
- [ ] **OAuth2 认证**
  - Bearer Token 验证（调用 account.hamr.store）
  - 令牌内省（Token Introspection）
  - 权限范围验证（Scope）
- [ ] **请求限流**
  - 基于 API Key 限流（默认 100 次/分钟）
  - 基于 IP 限流（防 DDoS）
  - 基于接口限流（热点接口保护）
  - 滑动窗口算法
  - 限流响应（429 Too Many Requests）
- [ ] **安全防护**
  - IP 黑白名单
  - 请求签名验证（HMAC-SHA256）
  - SQL 注入检测
  - XSS 攻击防护
  - CORS 配置

#### 3. API 版本管理
- [ ] **版本控制**
  - URL 路径版本化（/v1/xxx、/v2/xxx）
  - Header 版本化（Accept-Version）
  - 版本路由映射
- [ ] **向后兼容**
  - 旧版本保留（至少 6 个月）
  - 版本废弃通知
  - 自动重定向

#### 4. 可观测性
- [ ] **请求日志**
  - 请求 ID（UUID 追踪）
  - 请求/响应时间记录
  - 请求体/响应体（可选记录）
  - 调用者信息（API Key、IP、User-Agent）
- [ ] **性能监控**
  - QPS（每秒请求数）
  - 响应时间分布（P50、P90、P99）
  - 错误率统计
  - 上游服务健康状态
- [ ] **调用链追踪**
  - 分布式 Trace ID
  - 跨服务调用链路
  - 慢请求分析
- [ ] **告警通知**
  - 错误率超阈值告警
  - 响应时间超阈值告警
  - 服务不可用告警
  - 限流告警

#### 5. 错误处理
- [ ] **统一错误格式**
  ```json
  {
    "success": false,
    "error": {
      "code": "AUTH_001",
      "message": "API Key 无效或已过期",
      "details": "请检查您的 API Key 是否正确"
    },
    "request_id": "req_abc123"
  }
  ```
- [ ] **错误码体系**
  - `AUTH_xxx`：认证鉴权错误
  - `RATE_xxx`：限流错误
  - `ROUTE_xxx`：路由错误
  - `SVC_xxx`：上游服务错误
  - `SYS_xxx`：系统内部错误
- [ ] **熔断降级**
  - 上游服务不可用时返回降级响应
  - 熔断器（半开/全开/关闭）
  - 降级策略配置

#### 6. Webhook 回调
- [ ] **事件订阅**
  - 用户注册事件
  - 家庭创建事件
  - 数据变更事件
- [ ] **回调管理**
  - Webhook URL 注册
  - 签名验证（防伪造）
  - 重试机制（指数退避，最多 5 次）
  - 回调日志

#### 7. API 文档
- [ ] **OpenAPI 规范**
  - 自动生成 OpenAPI 3.0 文档
  - Swagger UI 交互式文档
  - API 变更日志
- [ ] **SDK 生成**
  - 基于 OpenAPI 自动生成 SDK
  - JavaScript/TypeScript SDK
  - Python SDK
  - Go SDK

### 服务边界与职责划分

#### API 网关负责
✅ 请求路由和转发
✅ 鉴权和限流
✅ 日志和监控
✅ API Key 管理
✅ 错误统一处理
✅ Webhook 回调

#### API 网关不负责
❌ 具体业务逻辑（由后端微服务实现）

❌ 用户认证逻辑（由 account-service 实现）
❌ 前端页面渲染

### 明确不包含

- 不包含：GraphQL 支持（Phase2）
- 不包含：WebSocket 长连接（Phase2）
- 不包含：gRPC 转换（Phase2）

## 🛠 技术方案

### 技术栈
- **API 网关**：Kong / APISIX / Envoy
- **后端语言**：Go / Node.js
- **数据库**：PostgreSQL（API配置）+ Redis（限流）
- **监控**：Prometheus + Grafana
- **日志**：ELK Stack
- **域名**：api.hamr.top
